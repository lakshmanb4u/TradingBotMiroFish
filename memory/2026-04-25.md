Key architecture decisions (validated against ChatGPT feedback):

- VWAP reclaim entry rule: only valid before 11:30 AM
- Score threshold: min 75 for production, relaxed to 60 for backtest

#### 2. Backtest Replay Engine (`services/earnings_sympathy/backtest_replay.py`)
Point-in-time replay with zero lookahead bias.
- `PointInTimeOHLCV` — hard-clips all bars to <= as-of date
- `PointInTimOptionsChain` — uses stored snapshots only (not live)
- Synthetic Black-Scholes chain builder when no snapshot exists
- `OutcomeEvaluator` — TP1 at 2x (50%), TP2 at 5x (30%), runner at 10x (20%), stop at -50%
- Outputs: backtest_report.md, .json, candidates.csv, option_outcomes.csv, skipped_candidates.csv

#### 3. UW Historical Fetcher (`services/earnings_sympathy/uw_historical_fetcher.py`)
Honest probe of what UW Basic API ($150/mo) actually supports historically.

CONFIRMED findings from live API probe:
- `/flow-alerts?date=` — date param silently ignored, returns current data
- `/flow-recent` — no date param, live only
- `/options-volume?date=` — date param silently ignored
- `/iv-rank` — returns last 5 days only
- `/darkpool?date=` — date param accepted, returns EMPTY for historical dates
- `/api/stock/{ticker}/flow-per-strike` — WORKS with date param ✅
- `/api/stock/{ticker}/oi-per-strike` — WORKS with date param ✅
- `/api/stock/{ticker}/iv-rank` — returns last 5 days
- `/api/historical/TICKER/YYYY-MM-DD` — "full tape" historical, returns market-closed error for non-trading-days

UW $250/mo tier required for full historical parquet data.
